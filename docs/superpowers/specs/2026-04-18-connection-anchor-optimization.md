# ConnectionPlugin 锚点优化 — 设计规格

---

## 一、概述

**优化目标**: 改进 `computeBezierPoints` 算法，使连线锚点选择基于实际距离计算，而非简单的四方向分支

**问题**: 当前实现仅根据中心点相对位置（左右/上下）选择锚点，对角线场景下可能选择非最优锚点

---

## 二、新算法

### 源元素锚点选择

源元素有 4 个锚点：

| 锚点 | x | y |
|------|---|---|---|
| top | 元素中心x | 元素顶部y |
| bottom | 元素中心x | 元素底部y |
| left | 元素左侧x | 元素中心y |
| right | 元素右侧x | 元素中心y |

### 锚点计算步骤

```
1. 计算目标元素中心点 (tgtCx, tgtCy)
2. 计算源元素 4 个锚点坐标
3. 计算每个锚点到目标中心的欧几里得距离
4. 选择距离最短的锚点作为起点
```

### 目标元素锚点选择

保持现有逻辑：根据相对位置选择对边中心点

| 关系 | 锚点 |
|------|------|
| 目标在源右 | 目标左边缘中点 |
| 目标在源左 | 目标右边缘中点 |
| 目标在源下 | 目标上边缘中点 |
| 目标在源上 | 目标下边缘中点 |

---

## 三、代码改动

**文件**: `src/app/plugins/utils/fabricPlugins/connectionPlugin.ts`

**方法**: `computeBezierPoints`

**改动内容**:
- 替换 4 分支 if-else 锚点选择逻辑
- 改为计算 4 个锚点到目标中心的距离，选择最近者

---

## 四、测试验证

手动测试场景：
1. 源在左，目标在右 → 选右锚点
2. 源在右上，目标在左下 → 选右上/下锚点（距离近者）
3. 源在上，目标在下 → 选下锚点
